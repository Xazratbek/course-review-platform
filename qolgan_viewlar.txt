“3 ta yangi feature”ni qanday implement qilish (kod bermasdan yo‘l-yo‘riq)
Saved Search & Smart Alerts

Frontendda “Save this filter” tugma qo‘ying (category/level/language/search).

Local storage’da saved presets saqlang (saved_searches array).

App ochilganda presetlarni o‘qing va periodik poll qiling (masalan 60s) getCourses()/getMyReviews() orqali.

Oxirgi ko‘rilgan natija hash/id snapshot bilan solishtirib “new match” bo‘lsa mavjud notification UI’ga local-inbox sifatida push qiling.

Keyin backend qo‘shilganda shu logikani server-side alertga migratsiya qilasiz.

Comparison Mode (Course vs Course)

Course listda checkbox/select state qo‘ying (selectedForCompare, max 3).

Compare panelda common field map chiqaring: price, rating, mentor, certificate, duration.

Bo‘sh fieldlar uchun — fallback va highlight best-value qo‘shing (masalan eng yuqori rating).

UX: sticky compare bar + “Open comparison” dedicated view.

Reputation & Trust Layer

Frontend scoring formula yarating (backendga tegmasdan):
trustScore = helpfulVotes * w1 + verifiedUser * w2 - reportRisk * w3.

verifiedUser ni profile flagdan, helpfulni vote summarydan, reportRiskni oddiy heuristikadan oling (masalan juda qisqa/repetitive text).

Review listda sorting option qo‘shing: “Most trusted”.

Har review kartasida trust badge + breakdown tooltip (nechaga shuncha ball bo‘ldi) chiqaring.
